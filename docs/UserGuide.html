1 Overview

What It Does

Day Trade Scanner Pro monitors the US equity market in real time, scores every symbol by a composite Trade Score, and surfaces the highest-probability trade candidates on your screen before the move is over. It replaces the manual process of scanning dozens of charts every morning.

The app connects to up to seven market-data providers simultaneously. If one fails or rate-limits, it automatically fails over to the next. Premium providers (Polygon.io, Finnhub) unlock live WebSocket price streaming; free providers (Yahoo Finance, Stooq) work with no configuration at all.

Feature Summary

🔍

Automated Scanning

Continuous or one-shot scans with configurable frequency (1 min → 1 hour). Results ranked by Trade Score, highest first.

📊

Trade Score 0–100

Composite score combining Momentum, Volume, Technical indicators, and News sentiment. Weights are fully adjustable.

⚡

Live WebSocket Streaming

Real-time trade ticks from Polygon.io and Finnhub. Prices update in the results table without re-scanning. Auto-reconnects on drops.

🔔

Alerts & Notifications

Price above/below, percent change, and volume spike alerts with sound and popup notifications.

📈

Candlestick Charts

Interactive OHLCV charts for any scanned symbol using LiveCharts2. Multiple timeframes supported.

💾

Export

One-click export to formatted .xlsx (color-coded scores, auto-filter) or plain .csv.

👁

Watchlist

Maintain a persistent list of symbols to monitor. Watchlist symbols are included in every scan run.

🔐

Encrypted Key Storage

API keys stored with Windows DPAPI — encrypted at rest, bound to your Windows account. Never stored in plaintext.

Installing the MSIX Package

  1. Download the installer

    Download TradeScanner_1.0.0.0.msix from the GitHub Releases page.

  2. Install the signing certificate (sideload only)

    If this is a self-signed build, install the certificate first. Open PowerShell as Administrator:

    Import-Certificate -FilePath TradeScanner-test.pfx -CertStoreLocation Cert:\LocalMachine\TrustedPeople
  3. Enable Developer Mode or sideloading

    Go to Settings → System → For developers and enable Sideload apps (Windows 10) or Developer Mode (Windows 11).

  4. Run the installer

    Double-click TradeScanner_1.0.0.0.msix. Click Install in the App Installer dialog.

  5. Launch the app

    Find Day Trade Scanner Pro in the Start menu. On first launch the database is created automatically.

Note The app stores its database at %LOCALAPPDATA%\TradeScanner\tradescanner.db. This file is created on first run and persists scan history, watchlists, alerts, and encrypted API keys.

Uninstalling

Go to Settings → Apps → Installed apps, find Day Trade Scanner Pro, and click Uninstall. User data (database file) at %LOCALAPPDATA%\TradeScanner\ is not removed automatically — delete that folder to fully clean up.

3 Quick Start

The fastest path to your first scan — no API key required (uses Yahoo Finance and Stooq).

  1. Launch Day Trade Scanner Pro

    The Dashboard opens by default. The status bar shows "Ready".

  2. Click "Scan Once"

    A single scan runs immediately against Yahoo Finance (always available, no key). Results appear in the table within 30–60 seconds depending on symbol list size.

  3. Review results

    Results are sorted by Trade Score (highest first). Each row shows symbol, score, price, change %, volume, and volume ratio.

  4. Click a symbol to open its chart

    Navigate to the Charts tab. The selected symbol's candlestick chart loads automatically.

  5. Add an API key for better data (optional)

    Go to Providers, paste your Polygon.io or Finnhub key, click Save. The provider becomes active immediately — no restart needed.

  6. Start continuous scanning

    Click the green Start Scan button. The scanner runs on the interval configured in Settings (default: 5 minutes).

Adding an API Key

  1. Navigate to Providers

    Click Providers in the left sidebar.

  2. Locate your provider

    Each provider appears as a card. A green dot means it is currently active; grey means unavailable (no key or key invalid).

  3. Paste your key

    Click the text field labeled "Paste API key here, then Save or Validate" and paste your key.

  4. Click Save

    The key is encrypted with Windows DPAPI and stored in the local database. The field clears and a 🔑 icon appears — confirming the key is saved.

  5. Optionally click Validate

    Validates the key by making a live test request to the provider. Shows "Valid ✓" or an error message in the status column.

To remove a key, click Clear. The provider reverts to unavailable until a new key is entered.

Failover Chain

The app uses an automatic failover chain ordered by priority:

  1. Polygon.io (priority 1, fastest)
  2. Finnhub (priority 2)
  3. Financial Modeling Prep (priority 3)
  4. TwelveData (priority 4)
  5. AlphaVantage (priority 5)
  6. Stooq (priority 6, daily only)
  7. Yahoo Finance (priority 10, always available)

If a provider returns an error or rate-limit response, the chain automatically retries with the next available provider. You do not need to configure this — it is automatic.

Yahoo Finance symbol universe — important limitation When Yahoo Finance is the only available provider (no API keys configured), the scanner operates on a fixed 30-symbol universe: AAPL, MSFT, NVDA, TSLA, META, AMZN, GOOGL, AMD, NFLX, CRM, PLTR, SOFI, RIVN, LCID, NIO, BABA, JD, SNAP, PINS, SPY, QQQ, IWM, DIA, GLD, SLV, USO, TLT, HYG, XLF. This is intentional — Yahoo Finance's public endpoint does not offer a "most active" list, so a curated day-trade universe is hardcoded. To scan the full market dynamically, add a premium API key (Polygon.io or Finnhub). The scan then requests up to 200 of the most active symbols from that provider's API.
Scan pipeline flow Each scan run: (1) requests up to 200 most-active symbols from the best available provider, (2) applies price / volume / change-percent filters, (3) fetches 50 bars of 5-minute historical data for up to 100 filtered candidates, (4) scores and ranks them, (5) returns the top N results (configured by Max Results). Only the top 100 candidates after filtering get their historical bars fetched — this keeps scan time reasonable on slower providers.

Real-Time Streaming

Streaming connects a WebSocket to Polygon.io or Finnhub and pushes live trade ticks directly into the results table — prices update without re-scanning.

  1. Select a streaming provider

    Use the dropdown in the toolbar (Polygon or Finnhub). Requires a saved API key for the selected provider.

  2. Optional: enter API key inline

    If you haven't saved a key via the Providers page, paste it in the streaming key field. If the field is left blank, the app silently uses the saved key.

  3. Click Stream

    The button label changes to Stop Stream. A "Connected" status appears. Prices in the table begin updating in real time.

  4. Click Stop Stream to disconnect

    The connection closes gracefully. Prices revert to scan-time values.

Reconnection If the WebSocket drops (network interruption, server restart), the app automatically reconnects with exponential backoff: 2s → 4s → 8s → 16s → 32s → 60s. The status bar shows "Connection lost — reconnecting in Xs (attempt N)…". No action is required.

Exporting Results

Click Export CSV in the Dashboard toolbar. A Save dialog opens with two file type options:

  • Excel Workbook (.xlsx) — Formatted spreadsheet with bold headers, color-coded Trade Score column (teal ≥ 70, amber 40–69, red < 40), color-coded Change % (teal = positive, red = negative), auto-filter on all columns, and auto-sized column widths.
  • CSV File (.csv) — Plain comma-separated values, compatible with any spreadsheet application.

Creating an Alert

  1. Navigate to Alerts

    Click Alerts in the left sidebar.

  2. Fill in the add-alert form

    Enter the ticker symbol (e.g., AAPL), choose an alert type from the dropdown, and enter the threshold value.

  3. Click Add Alert

    The alert appears in the list below, enabled by default.

  4. Manage existing alerts

    Use Toggle to temporarily enable/disable an alert without deleting it. Use Delete to remove it permanently.

The alert list columns show: Symbol, Type, Threshold, Enabled status, how many times it has triggered (Triggered), and when it last fired (Last At).

Notifications

When an alert fires, the app:

  • Plays a system sound (Asterisk) if Sound Alerts is enabled in Settings
  • Shows a popup notification if Popup Alerts is enabled in Settings

Both notification types can be toggled independently in Settings → Notifications.

7 Charts

The Charts page displays a candlestick chart for a selected symbol.

  1. Navigate to Charts

    Click Charts in the left sidebar.

  2. Enter a symbol

    Type a ticker symbol in the input field (e.g., TSLA) and select a timeframe from the dropdown (5m, 1h, 1d, 1w).

  3. Click Load

    The chart fetches historical bars from the active provider chain and renders a candlestick series. Green candles = close above open; red candles = close below open.

Timeframe availability Intraday timeframes (5m, 1h) require a premium provider (Polygon.io, Finnhub, FMP, or TwelveData). Stooq and Yahoo Finance only support daily (1d) and above. If no intraday data is available, the chart shows an empty state.

If no data has been loaded yet, the chart area displays "No chart data. Enter a symbol and click Load.". If a load is in progress, a loading spinner is shown instead.

8 Watchlist

The Watchlist lets you maintain a persistent list of symbols you always want scanned, regardless of the general filter criteria.

  1. Navigate to Watchlist

    Click Watchlist in the left sidebar.

  2. Add a symbol

    Type a ticker in the input field and press Enter or click Add to Watchlist. Duplicates are ignored silently.

  3. Refresh live quotes

    Click Refresh Quotes to pull the latest price, change %, and volume for every symbol on the list without running a full scan.

  4. Remove a symbol

    Click Remove on the right side of any row.

The watchlist table shows: Symbol, Price, Change% (green = positive, red = negative), Volume, and a Remove button per row.

Watchlist symbols are included in every scan run. They bypass the minimum price, volume, and change-percent filters defined in Settings, so they always appear in results.

Persistence Watchlist entries are stored in the local SQLite database and survive app restarts. They are device-specific (not synced across machines).

10 Security & Privacy

API Key Storage

API keys are encrypted using Windows Data Protection API (DPAPI) before being stored in the local SQLite database. DPAPI binds encryption to your Windows user account and machine — the encrypted blob cannot be decrypted on another machine or under a different Windows account, even if someone copies the database file.

What this means in practice If you reinstall Windows, restore your PC to factory settings, or move the database to another computer, saved API keys will not be recoverable. Re-enter them on the Providers page after re-installation.

Data Storage

All application data is stored locally on your device at:

%LOCALAPPDATA%\TradeScanner\
  tradescanner.db       — SQLite database (scan history, watchlists, alerts, encrypted keys)
  logs\                 — Application log files (Serilog, rotated daily)

No data is transmitted to any server other than the market data providers you have configured. The application makes only outbound HTTP/WebSocket requests to provider APIs.

11 Troubleshooting

The app won't install (MSIX)

Most common cause The signing certificate is not trusted. Run the certificate import command (see Installation, Step 2) from an elevated (Administrator) PowerShell session, then try installing again.

Scan returns no results

  • Check that the filter criteria in Settings are not too restrictive. Try lowering Min Trade Score to 0 and Min Change % to 0.
  • Verify that at least one provider is available (green dot on the Providers page). If all providers show grey, either no API keys are saved, or Yahoo Finance is failing — check your internet connection.
  • Check the status bar for any error message from the provider.

Streaming doesn't connect

  • Verify your Polygon.io or Finnhub key is saved on the Providers page (🔑 icon visible).
  • Premium WebSocket access may require a paid plan on the provider side. Finnhub's free tier supports WebSocket; Polygon.io requires at least a Starter plan.
  • Check for a firewall or VPN blocking outbound WSS connections to socket.polygon.io or ws.finnhub.io.

Charts show "No chart data"

  • Ensure you clicked Load after entering the symbol and timeframe.
  • If using a 5m or 1h timeframe, ensure a premium provider is configured — Stooq and Yahoo do not support intraday data.
  • Outside of US market hours (9:30 AM – 4:00 PM ET), intraday data for today may not yet be available.

API key shows "Invalid" after saving

  • Double-check that you copied the entire key with no leading/trailing spaces.
  • Some provider free tiers have daily request limits — the validate call may be failing due to rate limiting, not an invalid key.
  • If the key was saved from a different Windows account or PC, it will not decrypt correctly. Enter the key again.

Application logs

Detailed logs are written to %LOCALAPPDATA%\TradeScanner\logs\, one file per day. These are plain text files useful for diagnosing errors. If reporting a bug, please include the relevant log file.

13 Market-Hours Testing Checklist

Use this checklist during your first live market-hours session to confirm all subsystems are working correctly. US market hours: 9:30 AM – 4:00 PM Eastern Time.

Before Market Open (9:00–9:29 AM ET)

  1. Verify provider connectivity

    Navigate to Providers. Confirm at least one provider shows a green dot. Click Validate on your primary provider to confirm the API key is accepted.

  2. Check settings

    Go to Settings and confirm your filters are not too restrictive. For first-run testing, try: Min Price $5, Max Price $500, Min Volume 200,000, Min Change% 1.0, Min Score 40. Save settings.

  3. Add watchlist symbols

    Add 3–5 symbols you intend to monitor (e.g., AAPL, TSLA, NVDA) to the Watchlist. These will always appear in results regardless of filters.

  4. Confirm log file is being written

    Open %LOCALAPPDATA%\TradeScanner\logs\ and verify a log file exists for today's date.

At Market Open (9:30 AM ET)

  1. Run a single scan

    Click Scan Once and wait for it to complete. Confirm results appear in the table. Check the status bar shows "Scan complete — N results" with no error text.

  2. Verify Trade Scores make sense

    At open, momentum and volume are typically elevated. Expect scores of 50–80 for active names. A table full of 0s or all 100s suggests a data mapping issue — check the provider's response in the logs.

  3. Test streaming (if you have a premium key)

    Select Polygon or Finnhub in the streaming dropdown, leave the key field blank (uses saved key), click Stream. Watch the PRICE column for live updates. Prices should change within seconds for active symbols like AAPL.

  4. Test an alert

    Go to Alerts. Add a Price Above alert for AAPL at $1 (guaranteed to fire). Run a scan. Confirm the system sound plays and the Triggered counter increments. Delete the test alert afterward.

During Market Hours

  1. Start continuous scanning

    Click Start Scan. Confirm the status bar cycles through "Fetching symbols… → Scanning N symbols… → Ranking N candidates… → Scan complete — N results" on each interval.

  2. Monitor for rate-limit errors

    If the status bar shows provider errors or results drop to 0 unexpectedly, you may have hit an API rate limit. The failover chain will switch providers automatically. Check the logs to confirm which provider is being used.

  3. Test export

    Click Export CSV, choose Excel format, save the file, open it in Excel. Confirm the Trade Score column is color-coded and the Change% column shows green/red formatting.

  4. Test streaming reconnect

    With streaming active, temporarily disable your network adapter for 5 seconds, then re-enable it. The status bar should show "Connection lost — reconnecting in 2s (attempt 1)…" then "reconnected" within 10–15 seconds. No manual action required.

  5. View a chart

    Navigate to Charts, enter a symbol, select 5m timeframe, click Load. Confirm the candlestick chart renders with today's bars.

Known Behaviors (Not Bugs)

BehaviorExplanation
Scan takes 30–90 secondsThe scanner fetches up to 200 quotes then 50 bars × 100 symbols sequentially. This is expected, especially on slower free providers.

Chart shows no data for 5m/1h before 9:30 AM ETIntraday bars don't exist before market open. Use 1d timeframe for pre-market context.
